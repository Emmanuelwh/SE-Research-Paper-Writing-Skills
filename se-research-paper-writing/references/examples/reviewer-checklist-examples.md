# Reviewer Checklist Examples

## Claim-Evidence Map

`Claim: The tool improves vulnerability triage efficiency. | Evidence: Controlled task study with 24 participants, task completion time, and correctness. | Section/Table/Figure: Section 5.2, Table 3. | Threat: Participant sample is mostly graduate students. | Status: supported with scope limitation.`

## Reviewer-Risk Note

`Risk: The Introduction claims broad usefulness for industrial developers, but the Evaluation studies only open-source Java projects. Fix: scope the claim to Java open-source projects and add External Validity discussion.`

## RQ Alignment Map

`RQ: How accurately does the technique detect flaky tests? | Method: Evaluate on curated flaky-test benchmark. | Evidence: Precision, recall, F1 against three baselines. | Threat: Benchmark may underrepresent organization-specific flaky tests.`
